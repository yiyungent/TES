一个员工 所能评价的人 范围（评价人能评价哪些员工？）：
	根据部门和职位 共同作用
	
		系主任：【(评价人)】此员工 【所在系】 下 【所有教研室】 下 【所有职位】 的员工---根据部门筛选
				PS:递归，需要 评价人.Department 及其 评价人.Department.Children.....及其所有后代，直到.Children 为 null
				
		普通教师： 【(评价人)】此员工 【所在教研室】 下  【职位为  “普通教师”】 的员工---根据部门，职位 筛选
				PS: 为防止教研室下又有eg.小组，这里将其下面的小组也算入，如果不算入，则不需递归
				需要 评价人.Department 及其 评价人.Department.Children.....及其所有后代，直到.Children 为 null
		
		教研室主任： 【(评价人)】此员工 【所在教研室】下 【所有职位】 的员工----根据部门筛选
				PS: 为防止教研室下又有eg.小组，这里将其下面的小组也算入，如果不算入，则不需递归
				同样递归： 需要 评价人.Department 及其 评价人.Department.Children 及其所有后代，直到 .Department 为 null
				
		综上所述：
			可以发现，不管评价人是什么职位，第一次筛选都采用 从 评价人.Department 开始向下递归
				需要 评价人.Department 及其 评价人.Department.Children.....及其所有后代，直到.Children 为 null
			这便是根据评价人 进行部门筛选
				需要参数：评价人.Department
			
			
			第二次筛选：职位筛选：
				评价人为 “普通教师” 时，在 第一次已筛选的前提下，排除非 “普通教师”，即“普通教师” 只能评价 “普通教师”
